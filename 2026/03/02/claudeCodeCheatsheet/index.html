Claude Code Cheat Sheet

A quick reference for the Claude Code CLI. Covers the commands I use most, how to wire up hooks, define skills, and spin up subagents.

Slash Commands

Run these in the Claude Code prompt:

Command What it does
/help Show all available commands
/clear Clear conversation history
/compact Compact context to save tokens
/cost Show token usage and cost for the session
/status Show current model and config
/model Switch model
/memory View and manage persistent memory
/fast Toggle fast mode (Opus 4 with faster output)
/review-pr Review a pull request (if skill defined)
/commit Commit staged changes (if skill defined)

Slash commands prefixed with / that aren’t built-in are skills — more on those below.

Hooks

Hooks run shell commands automatically when Claude takes certain actions. Configure them in .claude/settings.json:

Common hook events

Event Fires when
user-prompt-submit-hook User sends a message
pre-tool-use Before any tool call
post-tool-use After any tool call
stop Claude finishes responding

Subagents (Task Tool)

Subagents let Claude spin up specialized agents for complex subtasks. Claude Code comes with several built-in types.

Built-in agent types

Type Best for
Bash Git operations, running commands
Explore Codebase search, finding files/patterns
Plan Designing implementation strategies
general-purpose Multi-step research tasks
claude-code-guide Questions about Claude Code itself

Basic usage (in a skill or prompt)

Claude calls the Task tool internally. You can instruct it explicitly:

“Use the Explore agent to find all API endpoint definitions, then use the Bash agent to run the tests.”

Key options

Option What it does
run_in_background: true Fire-and-forget; check output later
isolation: "worktree" Run in an isolated git worktree
resume: "<agent-id>" Resume a previous agent with its context

Parallel agents

When tasks are independent, Claude runs multiple agents at once:

“Search for all usages of getUser and simultaneously check for any open GitHub issues related to auth.”

This launches two agents in a single message — one Explore, one Bash — and merges the results.

Tips

  • Use /compact aggressively on long sessions — it preserves task context while freeing up the token window.
  • Skills + hooks together are powerful: a /deploy skill can trigger a post-hook that posts to Slack.
  • Worktree isolation (isolation: "worktree") is great for risky refactors — changes are thrown away if you decide not to keep them.
  • The Explore subagent is much cheaper on context than running many Grep/Glob calls yourself for open-ended searches.
