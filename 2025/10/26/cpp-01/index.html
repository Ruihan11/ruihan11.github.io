C++八股文01 - 基础语法与类型系统

Ruihan
2025-10-26

1. 基本数据类型

知识点

C++基本数据类型包括：整型、浮点型、字符型、布尔型

类型 大小 范围
int 4字节 -2^31 ~ 2^31-1
long long 8字节 -2^63 ~ 2^63-1
float 4字节 约7位精度
double 8字节 约15位精度
char 1字节 -128 ~ 127
bool 1字节 true/false

面试回答

Q: C++基本数据类型有哪些？各占多少字节？

A: C++基本类型包括：

  • 整型：int(4字节)、long long(8字节)、short(2字节)
  • 浮点型：float(4字节)、double(8字节)
  • 字符型：char(1字节)
  • 布尔型：bool(1字节)

需要注意的是，除了char固定1字节，其他类型大小在不同平台可能不同，可用sizeof查询。

2. 指针与引用

知识点

指针：存储变量地址的变量

引用：变量的别名

面试回答

Q: 指针和引用的区别？

A:

  1. 本质不同：指针是变量，引用是别名
  2. 能否为空：指针可以为nullptr，引用必须初始化且不能为空
  3. 能否重新赋值：指针可以改变指向，引用一旦绑定不能改变
  4. 内存占用：指针占用内存（通常4/8字节），引用不额外占用
  5. 使用方式：指针需解引用*ptr，引用直接使用

3. const与指针的组合

知识点

类型 语法 能否改值 能否改向
普通指针 int* p ✅ ✅
指向常量的指针 const int* p ❌ ✅
常量指针 int* const p ✅ ❌
指向常量的常量指针 const int* const p ❌ ❌

面试回答

Q: const int*、int* const、const int* const的区别？

A: 记忆技巧：从右往左读，看const在星号的哪边

  • const int* p：指向常量的指针，不能改值，可以改向
    • const在*左边，限制指向的内容
  • int* const p：常量指针，可以改值，不能改向
    • const在*右边，限制指针本身
  • const int* const p：两者都是常量，既不能改值，也不能改向

4. const与constexpr

知识点

特性 const constexpr
求值时机 运行时或编译时 编译时
使用场景 不可修改的变量 编译期常量、常量表达式
函数 不能修改成员变量 必须能在编译期计算

面试回答

Q: const和constexpr的区别？

A:

  1. 求值时机：

    • const：运行时或编译时均可
    • constexpr：必须在编译期确定

  2. 使用场景：

    • const：表示”不可修改”，用于变量、成员函数
    • constexpr：表示”编译期常量”，用于常量表达式、性能优化

  3. 函数差异：

    • const成员函数：不修改成员变量
    • constexpr函数：可在编译期执行，有严格限制

5. 左值与右值

知识点

左值（lvalue）：有名字，可以取地址，位于赋值号左边
右值（rvalue）：临时值，不能取地址，位于赋值号右边

面试回答

Q: 什么是左值和右值？有什么区别？

A:

  1. 定义：

    • 左值：有持久性，有名字，可以取地址（如变量）
    • 右值：临时性，无名字，不能取地址（如字面量、表达式结果）

  2. 引用绑定规则：

    • 左值引用T&：只能绑定左值
    • const左值引用const T&：可以绑定左值和右值
    • 右值引用T&&：只能绑定右值（C++11）

  3. 实际意义：

    • 右值引用是移动语义的基础
    • 延长临时对象生命周期
    • 避免不必要的拷贝

6. auto与decltype类型推导

知识点

auto推导规则：

  • 去掉顶层const和引用
  • auto&保留引用和const

decltype推导规则：

  • 保留类型完整信息
  • 括号改变结果

面试回答

Q: auto和decltype的区别？

A:

  1. 推导规则：

    • auto：类似模板推导，去掉顶层const和引用
    • decltype：保留表达式的完整类型信息

  2. 使用场景：

    • auto：简化变量声明，类型自动推导
    • decltype：需要精确类型信息，如返回类型推导

7. 四种类型转换

知识点

转换 用途 检查时机 安全性
static_cast 基础类型转换、类层次向上转换 编译期 较安全
dynamic_cast 多态类型的向下转换 运行期 安全
const_cast 去除const/volatile 编译期 危险
reinterpret_cast 指针类型转换、整数与指针互转 编译期 极危险

面试回答

Q: C++四种类型转换的区别和使用场景？

A:

  1. static_cast：

    • 编译期检查的类型转换
    • 用于：基本类型转换、类层次向上转换、void*转换
    • 不能去除const

  2. dynamic_cast：

    • 运行时类型检查，用于多态类型
    • 只能用于有虚函数的类
    • 失败返回nullptr（指针）或抛异常（引用）
    • 用于安全的向下转换

  3. const_cast：

    • 唯一能去除const/volatile的转换
    • ⚠️ 危险：修改原本const的对象是未定义行为
    • 合法场景：去除指向非const对象的const指针的const属性

  4. reinterpret_cast：

    • 重新解释比特位，不做任何检查
    • 用于：指针与整数互转、不同指针类型转换
    • ⚠️ 极度危险，破坏类型系统

8. static关键字

知识点

用法 作用域 生命周期 特点
static局部变量 函数内 程序结束 只初始化一次
static全局变量/函数 当前文件 程序结束 内部链接
static成员变量 类 程序结束 所有对象共享
static成员函数 类 - 不依赖对象，只能访问static成员

面试回答

Q: static关键字的作用？

A: static有四种用法：

  1. static局部变量：

    • 生命周期：整个程序运行期
    • 只初始化一次，保持值不变
    • 用于：计数器、缓存

  2. static全局变量/函数：

    • 作用域：限制在当前文件（内部链接）
    • 用于：防止命名冲突

  3. static成员变量：

    • 所有对象共享同一份
    • 必须在类外定义
    • 用于：对象计数、共享配置

  4. static成员函数：

    • 不依赖于对象，通过类名调用
    • 只能访问static成员
    • 没有this指针

9. inline关键字

知识点

三个作用：

  1. 建议编译器内联展开（可能被忽略）
  2. 允许在头文件中定义函数（避免重复定义）
  3. C++17：inline变量

面试回答

Q: inline关键字的作用？

A:

  1. 内联展开：建议编译器将函数调用处替换为函数体，减少函数调用开销

    • 编译器可能忽略（函数太大、递归等）
    • 现代编译器会自动内联优化

  2. 允许重复定义：可在头文件中定义，避免链接错误

    • 每个编译单元可见同一定义
    • 编译器保证只保留一份实现

  3. 类内定义自动inline：

    • 类内实现的成员函数自动是inline
    • 不需显式声明

  4. C++17 inline变量：

    • 允许在头文件中定义全局变量
    • 替代部分static的使用

注意：inline是建议而非强制，最终由编译器决定

10. extern关键字

知识点

作用：声明外部链接，引用其他文件定义的变量/函数

extern “C”：防止C++名称修饰，用于C/C++混合编程

面试回答

Q: extern关键字的作用？

A:

  1. 声明外部变量：

    • 告诉编译器变量定义在其他文件
    • 不分配内存，只是声明
    • 配合多文件编程

  2. 与static对比：

    • extern：外部链接，其他文件可见（默认）
    • static：内部链接，仅本文件可见

  3. extern “C”：

    • 防止C++名称修饰（name mangling）
    • 用于调用C库或被C调用
    • C++函数名会包含参数类型信息，C不会

总结：核心要点速记

概念 核心要点
指针vs引用 指针可为空可改向，引用是别名不能改
const指针 const在左→值不可改，const在右→向不可改
const vs constexpr const运行时，constexpr编译期
左值vs右值 左值有名可取址，右值临时不可址
auto 去顶层const和引用，简化声明
decltype 保留完整类型，括号变引用
static_cast 基础类型转换，编译期检查
dynamic_cast 多态向下转换，运行期检查，失败返nullptr
const_cast 去除const，危险操作
reinterpret_cast 重解释比特，极度危险
static局部 只初始化一次，生命周期到程序结束
static成员 所有对象共享，通过类名访问
inline 建议内联+允许重复定义
extern 声明外部链接，多文件共享
