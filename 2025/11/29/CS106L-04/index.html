第12讲：运算符重载 ➕➖×÷

为什么需要运算符重载？

“运算符能让类型行为像内置类型一样自然。”
——Bjarne Stroustrup

重载方式：成员 vs 非成员

方式 语法 适用场景 缺点
成员函数 bool operator<(const T& rhs) const; 修改左侧对象（如 +=, =） ❌ 左操作数必须是类类型
非成员函数 bool operator<(const T& lhs, const T& rhs); 对称操作（+, <, ==） ❌ 无法访问私有成员（除非 friend）

✅ STL 推荐：优先使用非成员重载（更泛化、支持隐式转换）。

示例：StanfordID 的 < 重载

🎯 最佳实践：

  • 对称操作（==, <, +）→ 非成员 + friend
  • 修改左侧（+=, =）→ 成员函数
  • 流操作（<<, >>）→ 必须非成员（ostream 不是你写的类！）

运算符重载常见陷阱 ⚠️

重载哲学：Principle of Least Astonishment (PoLA)

“不要让 + 做集合差，不要让 * 表示字符串拼接。”

运算符 推荐语义 禁忌
+, - 数学加减、集合并/差 字符串拼接（用 += 可接受）
<, > 严格弱序（strict weak ordering） 字典序以外的任意顺序（除非明确文档说明）
== 逐成员相等 引用相等（用 std::addressof 比较指针）
() 函数调用 任意副作用密集操作

总结与最佳实践

功能对比表

概念 核心要点 性能考量 适用场景
函数指针 传统、无状态 阻止内联 简单回调、C 兼容
Functor 有状态、类型安全 可内联优化 复杂逻辑、需要状态
Lambda 匿名 Functor 语法糖 同 Functor 一次性使用、简洁
泛型 Lambda auto 参数=模板 编译期多态 通用算法
std::function 类型擦除、运行时多态 虚表开销(~20ns) 存储/容器、动态绑定
成员重载 修改左侧 - +=, =, []
非成员重载 对称性、泛化 同成员 +, ==, <, <<
Spaceship <=> 自动生成 6 个比较(C++20) 零开销 所有可比较类型

实践指南

场景 推荐方案
一次性简单函数 ✅ Lambda（值捕获优先）
多次复用/复杂逻辑 ✅ 命名 Functor
泛型回调接口 ✅ 模板参数（template<typename F>）或 std::function（需类型擦除时）
对称运算（==, <） ✅ 非成员重载 + friend
修改左侧运算（+=） ✅ 成员重载
流操作（<<） ✅ 非成员重载（参数 ostream&, const T&）
避免歧义 ✅ 不要同时定义成员与非成员同名运算符
C++20 新项目 ✅ 优先使用 <=> 自动生成比较运算符
